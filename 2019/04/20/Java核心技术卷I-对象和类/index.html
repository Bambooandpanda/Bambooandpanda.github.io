Java核心技术卷I 对象和类

发表于 2019-04-20 | 分类于 Java学习

面向对象程序设计概述

对象

要使用OOP，一定要清楚对象的三个主要特征：

  • 对象的行为
    可以对对象施加哪些操作，或可以对对象施加哪些方法
  • 对象的状态
    当施加那些方法时，对象如何响应
  • 对象标识
    如何辨别具有相同行为与状态的不同对象

识别类

  • 识别类的简单规则是在分析问题的过程中寻找名词，而方法对应着动词。“找名词和动词”原则只是一种经验，在创建类的时候，哪些名词和动词是重要的完全取决于个人的开发经验

对象与对象变量

  • Java程序设计语言中，使用构造器构造新实例。
  • 如果需要的话，可以给对象传递一个方法，如System.out.println(new Date());或者，也可以将一个方法应用于刚刚创建的对象。如String s=new Date().toString()
  • 一定要认识到，一个对象变量并没有实际包含一个对象，而是仅仅引用了一个对象
  • 可以显式地将对象变量设置为null，表明这个对象变量目前没有引用任何对象
  • 如果将一个方法应用于一个值为null的对象上，那么就会产生运行时错误。

隐式参数与显式参数

  • 在每个方法中，关键字this表示隐式参数。有些程序员更偏爱这样的风格，因为这样可以将实例域与局部变量明显地区分开来。
  • 如果需要返回一个可变对象的引用，应该对它进行克隆。（因为前面说过，一个对象变量没有实际包含一个对象，而是仅仅引用了一个对象，如果返回的不是它的克隆，返回的引用的空间，就会破坏原来的封装性）

私有方法

  • 应该将所有的数据域都设置为私有的。
  • 有时候可能希望一个计算代码划分成若干的对立的辅助方法，通常，这些辅助方法不应该成为共有接口的的一部分，最好将这样的方法设计为private
  • 然而，只要方法是私有的，类的设计者就可以确信：它的不会被外部的其他类操作调用，可以将其删去，如果方法是公有的，就不能将其删去，因为其他代码很有可能依赖它

final

  • 可以将实例域定义为final。构建对象时必须初始化这样的域。也就是说，必须确保在一个构造器执行之后，这个域的值被设定，并且在后面的操作中，不能再对它进行修改。
  • final修饰符大都应用于==基本类型域==，或不可变的类域（如果类中的每个方法都不会改变其对象，这种类就是不可变的类）
  • final关键字只是表示存储在某个变量中的对象引用不会再指向其他对象。不过这个对象可以改变。

静态常量

静态方法

  • 静态方法是一种不能像对象是是操作的方法。例如Math.pow(a,b)方法就是一个静态方法。运算时不使用任何Math对象，换句话中，没有隐式的参数。可以认为静态方法是一种没有this的方法
  • 在下面两种情况下使用静态方法
    • 一个方法不需要访问对象状态，其所有参数都是通过显式参数提供
    • 一个方法只需要访问类的静态域

main方法

  • 每个类都可以有个main方法。这是一个常用于对类进行单元测试的技巧。如果想要独立地测试类，只需要执行java className

按值调用

  • Java程序设计语言总是采用按值调用。对象引用也是按值传递的
  • 总结一下Java中方法参数的使用情况
    • 一个方法不能修改一个基本数据类型的参数
    • 一个方法可以改变一个对象参数的状态
    • 一个方法不能让对象参数引用一个新的对象

重载

  • 重载和C++的重载相同

默认初始化

  • 如果在构造器中没有显式地给域赋予初值，那么就会自动地赋为默认值：数值为0，布尔值为false，对象引用为null。然而，只有缺少程序设计经验的人才会这么做。确实，如果不明确地对域进行初始化，就会影响程序代码的可读性。
  • 必须明确地初始化方法中的局部变量，但是如果没有初始化类中的域，将会被自动初始化为默认值

无参数的构造器

  • 如果在编写一个类的时候没有编写构造器，那么系统就会提供一个无参数的构造器。这个构造器将所有的实例域设置为默认值。
  • 如果类中提供了至少一个构造器，但是没有提供无参数的构造器，则在构造对象时如果没有提供参数就会被视为不合法。

显式域初始化

  • 与C++不同，可以在类定义中，直接将一个值赋给任何域。

参数名

  • 一个常用的技巧，在参数前加上a
  • 可以采用this的方法访问实例域

调用另一个构造器

  • 关键字this引用方法的隐式参数。然而，这个方法还有另外一个意义。如果构造器的第一个语句形如this（…），这个构造器将调用同一个类的另一个构造器。

初始化块

  • 初始化数据域的方法，Java还有第三种机制，称为初始化块。

    这种机制不是必须的，也不常见。通常会直接将初始化代码放在构造器中。

  • 调用构造器的具体处理步骤：

    • 所有数据域被初始化为默认值
    • 按照在类声明中出现的次序，依次执行所有域初始化语句和初始化块
    • 如果构造器第一行调用了第二个构造器，则执行第二个构造器主体
    • 执行这个构造器的主题

  • 如果对类的静态域进行初始化的代码比较复杂，那么可以使用静态的初始化块。将代码放在一个块中，并标记关键字static。

类的导入

  • 第一种方式
    在每个类名之前添加完整的包名

  • 第二种方式
    使用import语句，它应当位于源文件的顶部。但位于package的后面

    需要注意的是，只能使用星号导入一个包，不能使用import java.*或者import.java.*.*

    如果不同包的名称发生冲突，可以采用增加一个特定的import语句解决这个问题。import java.util.Date;如果这两个类都要使用，就在每个类名前面加上完整的包名。

静态导入

  • 静态导入就是导入静态方法和静态域。这样就可以直接使用类的静态方法和静态域，不用再加前缀了。

将类放入包中

要想将一个类放入包中，就必须将包的名字放在源文件的开头，包中定义类的代码之前。

如果没有在源文件中放置package语句，这个源文件中的类就被放置再一个默认包中。默认包是一个没有名字的包。将包中的文件放到与完整的包名匹配的子目录中。例如com.hosrtmann.corejava包中的所有源文件应该被放置在子目录com//hosrtman//corejava中。编译器也将类文件放在相同的目录结构中。想要编译一个程序，只需要改变基目录.

包作用域

  • 如果在定义类的时候没有指定public或者private，这个部分可以被同一个包中的所有方法访问。

文档注释

  • @para 变量描述
  • @return 描述
  • @throws 类描述
  • @author 姓名
  • @version 版本
  • @since 这里的text可以是对引入特性的版本描述
  • @deprecated 标记将对类，方法或者变量添加一个不再使用的注释

包与概述注释

  • 用/* /文档注释界定就可以了。但是，如果要产生包注释，就需要在每个包目录中添加一个单独的文件，有如下两种方法

    • 提供一个package.html的HTML文件
    • 提供一个package-info.java的文件。这个文件必须包含一个初始的/* /界定的javadoc注释。它不应该包含更多的代码或者注释

    类设计技巧

    • 一定要保证数据私有
      这是最重要的

    • 一定要对数据进行初始化
      Java不对局部变量进行初始化，但是会对对象的实例域进行初始化，最好不要依赖于系统的默认值

    • 不要再类中使用过多的基本类型
      这就是说，用其他的类代替多个相关的基本类型的使用

      这就可以用一个称为Address的类替换上面的实例域。

    • 不是所有的域都要独立的域访问器和更改器
      比如那些应当一旦设定不能更改的实例域。

    • 将职责过多的类进行分解

    • 类名和方法名要体现他们的职责

    • 优先使用不可变的类
      因为这样，如果多个线程试图同时更新一个对象，就会发生并发更改，这样的结果是不可预料的。如果类是不能改变的。就可以安全的在多个线程间共享对象。
